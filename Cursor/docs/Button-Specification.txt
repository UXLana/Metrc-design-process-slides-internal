================================================================================
OVERVIEW
================================================================================

The Button component allows users to take actions and make choices with a 
single tap. Buttons communicate actions that users can take throughout the UI.

Key Features:
- Two sizes: Large (48px) and Medium (36px)
- Three emphasis levels: High, Mid, Low
- Four states: Enabled, Hover, Pressed, Disabled
- Icon support: Left icon, Right icon, Icon-only
- Dark surface support
- Destructive variant for dangerous actions
- Button groups for related actions

================================================================================
SIZE SPECIFICATIONS
================================================================================

LARGE (lg)
----------
- Height: 48px
- Min Width: 100px
- Padding X: 16px
- Padding Y: 12px
- Icon Size: 24px
- Gap (between elements): 4px

Usage: Used at the bottom of forms or areas where major prominence is needed.

MEDIUM (md)
-----------
- Height: 36px
- Min Width: 80px
- Padding X: 12px
- Padding Y: 8px
- Icon Size: 20px
- Gap (between elements): 4px

Usage: Used in navigation bars, snackbars, alert dialogs, banners, and 
mid-page content areas.

ICON-ONLY BUTTONS
-----------------
Large Icon-Only: 40px × 40px
Medium Icon-Only: 32px × 32px

================================================================================
BUTTON GROUP SPACING
================================================================================

DEFAULT SPACING
---------------
- Gap between buttons: 8px
- Bottom margin: 32px
- Usage: Standard button groups in dialogs and forms

FORM SPACING
------------
- Gap between buttons: 16px
- Bottom margin: 32px
- Usage: Form submission areas

FORM MOBILE SPACING
-------------------
- Direction: Column (stacked vertically)
- Gap between buttons: 16px
- Bottom margin: 16px
- Usage: Mobile form layouts with full-width buttons

INLINE SPACING
--------------
- Gap between buttons: 8px
- Bottom margin: 0px
- Usage: Inline actions in tables, lists, cards

COMPONENT PROPS (TypeScript Interface)
================================================================================

interface ButtonProps {
  // Size of the button
  size?: 'lg' | 'md'  // Default: 'md'
  
  // Emphasis level of the button
  emphasis?: 'high' | 'mid' | 'low'  // Default: 'high'
  
  // Whether button is displayed on a dark surface
  onDark?: boolean  // Default: false
  
  // Whether this is a destructive action
  destructive?: boolean  // Default: false
  
  // Icon to display before the text (left side)
  leftIcon?: ReactNode
  
  // Icon to display after the text (right side / dropdown caret)
  rightIcon?: ReactNode
  
  // Whether this is an icon-only button (no text)
  iconOnly?: boolean  // Default: false
  
  // Loading state
  loading?: boolean  // Default: false
  
  // Full width button
  fullWidth?: boolean  // Default: false
  
  // Shows focus ring (for demonstration)
  focused?: boolean  // Default: false
  
  // Standard button HTML attributes
  disabled?: boolean
  type?: 'button' | 'submit' | 'reset'
  onClick?: (event: MouseEvent) => void
  // ... extends React.ButtonHTMLAttributes<HTMLButtonElement>
}

interface ButtonGroupProps {
  // Child buttons
  children: ReactNode
  
  // Spacing variant
  spacing?: 'default' | 'form' | 'formMobile' | 'inline'  // Default: 'default'
  
  // Alignment
  align?: 'start' | 'center' | 'end' | 'stretch'  // Default: 'start'
}

================================================================================
USAGE EXAMPLES
================================================================================

BASIC USAGE
-----------
import { Button } from '@/components'

// Primary action
<Button emphasis="high" size="lg">
  Submit
</Button>

// Secondary action
<Button emphasis="mid" size="md">
  Cancel
</Button>

// Tertiary action
<Button emphasis="low" size="md">
  Learn More
</Button>

WITH ICONS
----------
// Left icon
<Button leftIcon={<IconPlus />}>
  Add Item
</Button>

// Right icon (dropdown)
<Button rightIcon={<DropdownIcon />}>
  Options
</Button>

// Icon only (always include aria-label!)
<Button iconOnly leftIcon={<IconSettings />} aria-label="Settings" />

DESTRUCTIVE ACTIONS
-------------------
<Button emphasis="high" destructive>
  Delete
</Button>

<Button emphasis="mid" destructive>
  Remove
</Button>

ON DARK SURFACES
----------------
<Button emphasis="high" onDark>
  Action
</Button>

BUTTON GROUPS
-------------
import { Button, ButtonGroup } from '@/components'

// Form actions
<ButtonGroup spacing="form">
  <Button emphasis="low">Cancel</Button>
  <Button emphasis="high">Save</Button>
</ButtonGroup>

// Mobile stacked
<ButtonGroup spacing="formMobile" align="stretch">
  <Button emphasis="high">Continue</Button>
  <Button emphasis="low">Go Back</Button>
</ButtonGroup>

LOADING STATE
-------------
<Button emphasis="high" loading>
  Saving...
</Button>

================================================================================
DESIGN GUIDANCE
================================================================================

WHEN TO USE EACH SIZE
---------------------
Large (lg):
  - Bottom of forms
  - Areas where major prominence is needed
  - Primary calls-to-action
  - Full-page layouts

Medium (md):
  - Navigation bars
  - Snackbars
  - Alert dialogs
  - Banners
  - Mid-page content areas
  - Tables and lists

WHEN TO USE EACH EMPHASIS
-------------------------
High (Primary):
  - Most important action on screen
  - Primary calls-to-action
  - Form submission
  - Limit: One per section/viewport

Mid (Secondary):
  - Alternative options
  - Secondary actions
  - Filter/sort controls
  - Can use multiple

Low (Tertiary):
  - Least prominent actions
  - Cancel buttons
  - "Learn more" links
  - Can use multiple

BUTTON ORDERING
---------------
Always place low emphasis buttons before high emphasis buttons to create
proper visual hierarchy:

✓ Correct: [Cancel] [Save]
✗ Wrong:   [Save] [Cancel]

ICON GUIDELINES
---------------
- Left icons indicate what the action does (Add, Edit, Delete)
- Right icons indicate additional options (dropdown, menu)
- Icon-only buttons MUST have aria-label for accessibility
- Keep icon size consistent with button size (24px for lg, 20px for md)

DESTRUCTIVE ACTIONS
-------------------
- Use destructive styling for actions that delete data
- Use for irreversible or dangerous operations
- Consider using confirmation dialogs before destructive actions
- Only high and mid emphasis levels support destructive variant

SPACING IN DIALOGS
------------------
- Use default or form spacing in dialogs
- Primary action should be on the right
- Always provide a cancel/close option

MOBILE CONSIDERATIONS
---------------------
- Use formMobile spacing for stacked button layouts
- Consider full-width buttons on mobile
- Ensure adequate touch target size (minimum 48px)

================================================================================
ACCESSIBILITY
================================================================================

- All buttons are keyboard accessible (Tab to focus, Enter/Space to activate)
- Focus ring is visible and meets WCAG contrast requirements
- Icon-only buttons require aria-label
- Disabled buttons should have proper disabled styling
- Loading buttons should indicate loading state to screen readers
- Button groups should be wrapped in role="group"
