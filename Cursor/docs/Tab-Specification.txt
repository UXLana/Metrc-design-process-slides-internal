=============================================================================
TAB COMPONENT SPECIFICATION
Metrc Design System v1.0.0
=============================================================================

OVERVIEW
--------
Tabs organize content into separate views where only one view can be visible
at a time. They enable users to switch between related content quickly without
navigating to different pages.


=============================================================================
COMPONENT ANATOMY
=============================================================================

┌─────────────────────────────────────────────────────────────────────────┐

=============================================================================
USAGE EXAMPLES
=============================================================================

BASIC USAGE
-----------
```tsx
import { TabBar } from '@/components'

const [activeTab, setActiveTab] = useState('1')

const tabs = [
  { id: '1', label: 'Overview' },
  { id: '2', label: 'Details' },
  { id: '3', label: 'Settings' },
]

<TabBar
  tabs={tabs}
  activeTab={activeTab}
  onTabChange={setActiveTab}
/>
```


WITH ICONS (LEADING)
--------------------
```tsx
const tabsWithIcons = [
  { id: '1', label: 'Home', icon: <IconHome /> },
  { id: '2', label: 'Analytics', icon: <IconChart /> },
  { id: '3', label: 'Settings', icon: <IconSettings /> },
]

<TabBar
  tabs={tabsWithIcons}
  activeTab={activeTab}
  onTabChange={setActiveTab}
  iconPosition="leading"
/>
```


WITH ICONS (TOP)
----------------
```tsx
<TabBar
  tabs={tabsWithIcons}
  activeTab={activeTab}
  onTabChange={setActiveTab}
  iconPosition="top"
/>
```


WITH BADGES
-----------
```tsx
const tabsWithBadges = [
  { id: '1', label: 'All Items' },
  { id: '2', label: 'Pending', badge: 12 },
  { id: '3', label: 'Notifications', badge: 99 },
]

<TabBar
  tabs={tabsWithBadges}
  activeTab={activeTab}
  onTabChange={setActiveTab}
/>
```


STRETCHED TABS
--------------
```tsx
<TabBar
  tabs={tabs}
  activeTab={activeTab}
  onTabChange={setActiveTab}
  align="stretch"
  stretched
/>
```


ON DARK SURFACE
---------------
```tsx
<div style={{ background: '#13352C', padding: '24px' }}>
  <TabBar
    tabs={tabs}
    activeTab={activeTab}
    onTabChange={setActiveTab}
    onDark
  />
</div>
```


SCROLLABLE TABS
---------------
```tsx
const manyTabs = [
  { id: '1', label: 'Overview' },
  { id: '2', label: 'Details' },
  { id: '3', label: 'Analytics' },
  { id: '4', label: 'Settings' },
  { id: '5', label: 'Activity Log' },
  { id: '6', label: 'Notifications' },
  { id: '7', label: 'Permissions' },
]

<TabBar
  tabs={manyTabs}
  activeTab={activeTab}
  onTabChange={setActiveTab}
  scrollable
/>
```


=============================================================================
DESIGN GUIDELINES
=============================================================================

WHEN TO USE TABS
----------------
✓ Organize related content that doesn't need to be viewed simultaneously
✓ Allow users to quickly switch between views without page navigation
✓ Group settings, content types, or data categories
✓ Maintain context while exploring different aspects of content


BEST PRACTICES
--------------
DO                                      DON'T
────────────────────────────────────────────────────────────────────────────
Keep labels short (1-2 words)           Use long sentences as labels
Use icons consistently across tabs      Mix tabs with and without icons
Limit to 2-7 tabs; scroll for more     Show too many tabs without scrolling
Use badges sparingly                    Add badges to every tab
Place primary/most used tab first       Randomly order tabs
Maintain consistent icon position       Change icon positions mid-tab bar


ICON POSITION GUIDELINES
------------------------
Position    Use Case
────────────────────────────────────────────────────────────────────────────
none        Default for text-based navigation
leading     When icons help identify content quickly
top         Mobile layouts or when icons need prominence
only        Toolbar navigation with universally recognized icons


MINIMUM TAB WIDTH
-----------------
- Ensure tabs have adequate touch targets (minimum 44px)
- Don't compress labels to fit more tabs
- Use scrollable mode for many tabs


ACCESSIBILITY
-------------
- Tabs use role="tab" and aria-selected for screen readers
- TabBar uses role="tablist" 
- Keyboard navigation with arrow keys is recommended
- Focus states are clearly visible (#3086BF ring)
- Color contrast meets WCAG AA standards
