=============================================================================
ICON SPECIFICATION
Metrc Design System v1.0.0
=============================================================================

ICON STYLE: Outline/Stroke-Based (Similar to Heroicons/Feather)

This specification ensures visual consistency when generating icons for the
Metrc Design System. All icons should follow these guidelines.


=============================================================================
VISUAL SPECIFICATIONS
=============================================================================

STROKE PROPERTIES
-----------------
Property            Value               Notes
────────────────────────────────────────────────────────────────────────────
Stroke Width        2px                 Consistent across all icons
Stroke Linecap      round               Rounded line endings
Stroke Linejoin     round               Rounded corners at joints
Fill                none                Icons are outline-only, never filled
Stroke Color        currentColor        Inherits from parent text color


VIEWBOX SIZES
-------------
Size        ViewBox         Use Case
────────────────────────────────────────────────────────────────────────────
Standard    0 0 24 24       Default size for most icons
Compact     0 0 20 20       Smaller UI elements, badges, compact buttons


OPTICAL SIZING
--------------
- Icons should be centered within the viewBox
- Leave ~2px padding from edges for visual balance
- Actual drawn content typically occupies 20x20 within 24x24 viewBox
- Simple icons (arrows, plus) can extend closer to edges


=============================================================================
REACT COMPONENT STRUCTURE
=============================================================================

TEMPLATE
--------
```tsx
function Icon{Name}({ 
  size = 24, 
  color = 'currentColor' 
}: { 
  size?: number
  color?: string 
}) {
  return (
    <svg
      width={size}
      height={size}
      viewBox="0 0 24 24"
      fill="none"
      xmlns="http://www.w3.org/2000/svg"
    >
      <path
        d="[PATH_DATA]"
        stroke={color}
        strokeWidth="2"
        strokeLinecap="round"
        strokeLinejoin="round"
      />
    </svg>
  )
}
```

REQUIRED ATTRIBUTES
-------------------
- width={size}           Dynamic sizing
- height={size}          Maintains square aspect ratio
- viewBox="0 0 24 24"    Consistent coordinate system
- fill="none"            No fill - outline only
- xmlns                  SVG namespace (for compatibility)


PATH ATTRIBUTES
---------------
- stroke={color}         Uses color prop
- strokeWidth="2"        Consistent 2px stroke
- strokeLinecap="round"  Rounded line ends
- strokeLinejoin="round" Rounded corners

=============================================================================
DESIGN PRINCIPLES
=============================================================================

1. SIMPLICITY
   - Use minimum paths to convey meaning
   - Avoid unnecessary details
   - Icons should be recognizable at small sizes (16-20px)

2. CONSISTENCY
   - Same stroke weight across all icons
   - Same corner rounding (strokeLinejoin)
   - Same line endings (strokeLinecap)
   - Consistent visual weight/density

3. CLARITY
   - Clear silhouette even at small sizes
   - Distinct from similar icons
   - Universal recognition where possible

4. ALIGNMENT
   - Center icons optically (not mathematically)
   - Account for visual weight distribution
   - Pointed shapes may need slight offset


=============================================================================
DO'S AND DON'TS
=============================================================================

DO
──────────────────────────────────────────────────────────────────────────
✓ Use stroke-based outlines only
✓ Keep strokeWidth="2" consistent
✓ Use round linecaps and linejoins
✓ Keep paths simple and clean
✓ Center icons within viewBox
✓ Use currentColor for inheritance
✓ Test at multiple sizes (16px, 20px, 24px, 32px)
✓ Use single continuous paths where possible

DON'T
──────────────────────────────────────────────────────────────────────────
✗ Use filled shapes (fill attribute)
✗ Mix stroke widths within an icon
✗ Use square linecaps or mitered joins
✗ Add decorative flourishes
✗ Use gradients or multiple colors
✗ Create overly complex paths
✗ Hardcode colors (always use color prop)
✗ Use transforms in the SVG (bake into paths)


=============================================================================
GENERATION PROMPT TEMPLATE
=============================================================================

When generating new icons, use this prompt structure:

```
Generate a React SVG icon component for: [DESCRIPTION]

Requirements:
- Outline/stroke style only (no fills)
- ViewBox: 0 0 24 24
- Stroke width: 2px
- Stroke linecap: round
- Stroke linejoin: round
- Color prop with 'currentColor' default
- Size prop with 24 default
- Simple, clear paths that work at 16px-32px sizes
- Follow Heroicons/Feather icon style

Component structure:
function Icon{Name}({ size = 24, color = 'currentColor' }) {
  return (
    <svg width={size} height={size} viewBox="0 0 24 24" fill="none">
      <path d="..." stroke={color} strokeWidth="2" strokeLinecap="round" strokeLinejoin="round" />
    </svg>
  )
}
```
